Image sets:

First, you need to define the image sets you would like to compare.

Each set is defined by a pair (directory,suffix), where the directory is the input directory to look for .json or image files (usually jpeg or png files), and the suffix is the suffix at the end of the file used to define the set.

You have several use cases:

  • A json configuration file is available for your outputs, this file is usually in the directory config of the application but could also be anywhere. The configuration file will define the input and output of your algorithm/process, and each a json file is expected for each result. The json file of each results contains information that can later be used in the table sheet display, and it also contains the list of output images related to this result in a field named 'resulting_files'.

 

You can use a json configuration file to parameter the interface. In this case, the program will look for .json files in the input directories.

 

Key events:

Key_N is the one of the keys 0 to 9

ALT+Key_N switch temporarily to resulting image number N while the key is pressed
ALT+CTL+Key_N switch to image number N as the reference image
CTL+Key_N computes and displays difference between reference image and resulting image number N
ALT++ move to next row (seems broken)
ALT+- move to previous row (seems broken)
ALT+A: 
select upper left quarter crop
ALT+B 
select upper right quarter crop
ALT+C 
select lower left quarter crop
ALT+D select lower right quarter crop
ALT+V 
Vertical crop top: Horizontal reduction by 2
ALT+H 
Horizontal crop: Vertical reduction by 2
ALT+F 
 select full image geometry
CTL+C  copy text from the table
F1  launch beyond compare on results
F3  launch  faststone on results

 

 

 Note: All the features are not necessarily available with the pyqtgraph option at the moment.
